### ?? 3. The Game Phases (State Machine)

The Durable Object manages the game as a strict state machine to prevent desyncs between Three.js clients.

1. **Phase 1: Penggal (Session Start)**
   * Players join via WebSocket. Empty slots are assigned to AI Coalitions.
2. **Phase 2: Undi (Roll Dice)**
   * Current player rolls. Three.js animates. Server calculates target tile.
3. **Phase 3: Dewan (Action Resolution)**
   * Evaluate tile: Buy, Pay Rent, or Draw Card.
   * *If AI turn:* Server pauses, calls GLM 5.2, waits for AI decision, executes.
4. **Phase 4: Krisis (Event Phase)**
   * If doubles rolled, trigger GLM 5.2 to generate a "Krisis Nasional" based on real-time market data.
5. **Phase 5: Pembubaran (End Game)**
   * Player reaches 112 Seats (Properties) or bankrupts others.

### ?? 4. GLM 5.2 Agent Integration & Prompt Engineering

The AI acts as both the "Speaker of the House" (Narrator) and the independent political candidates. We use a structured system prompt to enforce JSON output, making it easy for the Durable Object to parse the AI's decisions.

```javascript
// backend/src/ai-agent.js

const GLM_SYSTEM_PROMPT = `
You are the Game Master and AI Political Agent for "Dewan Rakyat", a Malaysian Political Monopoly game.
You simulate cynical, satirical Malaysian politicians. You speak in a mix of formal Malay and Manglish.

CURRENT WORLD STATE:
- Market (PasarAPI): KLCI is {klcChange}%, CPO is RM{cpoPrice}.
- Seat Demographics (Tindak Malaysia): {demographicBreakdown}

YOUR ROLE:
1. If asked to decide a move for an AI player, respond with a JSON object.
2. Evaluate the property cost vs AI's current votes. Consider demographic advantages.
3. Provide a 1-sentence satirical quote in Manglish/Malay.

RESPONSE FORMAT (Strict JSON):
{
  "action": "BUY" | "IGNORE" | "TRADE",
  "quote": "Your satirical quote here.",
  "reasoning": "Brief internal logic."
}
`;

export async function getAIMove(gameState, env) {
  const marketData = JSON.parse(await env.MARKET_KV.get("market_state"));
  
  const promptPayload = {
    klcChange: marketData.klciChange,
    cpoPrice: marketData.cpo,
    demographicBreakdown: gameState.currentTile.demographics,
    aiVotes: gameState.currentPlayer.votes,
    tilePrice: gameState.currentTile.price
  };

  const response = await fetch(env.GLM_API_ENDPOINT, {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${env.GLM_API_KEY}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      model: "glm-5.2", // Target GLM 5.2
      messages: [
        { role: "system", content: GLM_SYSTEM_PROMPT.replace(/{(\w+)}/g, (_, k) => promptPayload[k] || '') },
        { role: "user", content: `You landed on ${gameState.currentTile.name}. Do you buy it?` }
      ],
      temperature: 0.7 // Slightly creative but logical
    })
  });

  const data = await response.json();
  
  try {
    // Extract JSON from GLM response
    const content = data.choices[0].message.content;
    const jsonStart = content.indexOf('{');
    const jsonEnd = content.lastIndexOf('}');
    return JSON.parse(content.substring(jsonStart, jsonEnd + 1));
  } catch (e) {
    // Fallback if GLM hallucinates
    return { action: "IGNORE", quote: "Macham, saya kenot brain sekarang.", reasoning: "Parse error" };
  }
}
```

### ?? 5. Authoritative Game State (Durable Object)

This is the brain of the multiplayer game. It handles WebSocket connections, processes turns, and calls the AI agent.

```javascript
// backend/src/GameDurable.js

import { getAIMove } from './ai-agent.js';

export class GameDurable {
  constructor(state, env) {
    this.state = state;
    this.env = env;
    this.sessions = new Map(); // WebSocket clients
  }

  async fetch(request) {
    const pair = new WebSocketPair();
    const [client, server] = pair;

    this.handleSession(server);
    return new Response(null, { status: 101, webSocket: client });
  }

  async handleSession(websocket) {
    this.state.acceptWebSocket(websocket);
    this.sessions.add(websocket);

    websocket.addEventListener('message', async (event) => {
      const msg = JSON.parse(event.data);
      
      if (msg.type === 'ROLL_DICE') {
        await this.processTurn(msg.playerId);
      }
    });
  }

  async broadcast(message) {
    const msgStr = JSON.stringify(message);
    for (const ws of this.sessions) {
      ws.send(msgStr);
    }
  }

  async processTurn(playerId) {
    const dice1 = Math.floor(Math.random() * 6) + 1;
    const dice2 = Math.floor(Math.random() * 6) + 1;
    const total = dice1 + dice2;
    
    // Update position
    const player = this.gameState.players[playerId];
    player.position = (player.position + total) % 40;
    const tile = this.gameState.board[player.position];

    await this.broadcast({ type: 'DICE_ROLLED', dice: [dice1, dice2], playerId });

    // If player is AI, ask GLM 5.2 what to do
    if (player.isAI) {
      // Broadcast "AI is thinking..."
      await this.broadcast({ type: 'AI_THINKING', playerId });
      
      const aiDecision = await getAIMove({
        currentPlayer: player,
        currentTile: tile
      }, this.env);

      if (aiDecision.action === 'BUY' && player.votes >= tile.price) {
        player.votes -= tile.price;
        tile.owner = playerId;
      }

      await this.broadcast({ 
        type: 'AI_ACTION', 
        playerId, 
        action: aiDecision.action, 
        quote: aiDecision.quote 
      });
    }

    // Check for doubles or pass to next player
    if (dice1 !== dice2) {
      this.gameState.currentTurn = (this.gameState.currentTurn + 1) % this.gameState.players.length;
    }

    await this.broadcast({ type: 'STATE_UPDATE', state: this.gameState });
  }
}
```

### 1. The 3D Board Generator (`scene/BoardBuilder.js`)

This module calculates the (x, z) coordinates for all 40 spaces in a square loop and renders them as 3D meshes.

```javascript
import * as THREE from 'three';

// The 40 spaces (simplified for brevity, mapping to previous JSON)
const BOARD_SPACES = [
  { id: 0, name: 'Pilihan Raya', type: 'go', color: 0x00FF00 },
  { id: 1, name: 'MUDA', type: 'property', color: 0x8B4513 },
  // ... spaces 2 to 38 ...
  { id: 39, name: 'Kuala Lumpur', type: 'property', color: 0x000080 }
];

export class BoardBuilder {
  constructor(scene) {
    this.scene = scene;
    this.tiles = [];
    this.tileSize = 4; // Size of each square
    this.boardWidth = 11 * this.tileSize; // 11 tiles per side
  }

  build() {
    const boardGroup = new THREE.Group();

    BOARD_SPACES.forEach((space, index) => {
      const tileMesh = this.createTile(space, index);
      boardGroup.add(tileMesh);
      this.tiles.push(tileMesh);
    });

    // Add center base
    const baseGeo = new THREE.BoxGeometry(this.boardWidth, 0.5, this.boardWidth);
    const baseMat = new THREE.MeshStandardMaterial({ color: 0x1a1a1a });
    const base = new THREE.Mesh(baseGeo, baseMat);
    base.position.y = -0.3;
    boardGroup.add(base);

    this.scene.add(boardGroup);
    return this.tiles;
  }

  // Calculate (x, z) for a tile in a square loop
  getTilePosition(index) {
    const step = this.tileSize;
    const halfBoard = this.boardWidth / 2;
    const halfStep = step / 2;
    const tilesPerSide = 10; // 10 non-corner tiles per side + 4 corners = 11 per side

    let x, z;

    if (index < tilesPerSide + 1) {
      // Side 1: Right side (Bottom to Top) - Spaces 0 to 10
      x = halfBoard - halfStep;
      z = halfBoard - halfStep - (index * step);
    } else if (index < (tilesPerSide * 2) + 1) {
      // Side 2: Top side (Right to Left) - Spaces 11 to 20
      x = halfBoard - halfStep - ((index - (tilesPerSide + 1)) * step);
      z = -halfBoard + halfStep;
    } else if (index < (tilesPerSide * 3) + 1) {
      // Side 3: Left side (Top to Bottom) - Spaces 21 to 30
      x = -halfBoard + halfStep;
      z = -halfBoard + halfStep + ((index - (tilesPerSide * 2 + 1)) * step);
    } else {
      // Side 4: Bottom side (Left to Right) - Spaces 31 to 39
      x = -halfBoard + halfStep + ((index - (tilesPerSide * 3 + 1)) * step);
      z = halfBoard - halfStep;
    }

    return { x, z };
  }

  createTile(space, index) {
    const pos = this.getTilePosition(index);
    
    const geometry = new THREE.BoxGeometry(this.tileSize * 0.95, 0.2, this.tileSize * 0.95);
    const material = new THREE.MeshStandardMaterial({ 
      color: space.color || 0x333333,
      roughness: 0.6 
    });
    
    const tile = new THREE.Mesh(geometry, material);
    tile.position.set(pos.x, 0, pos.z);
    tile.receiveShadow = true;
    tile.userData = { spaceId: index, name: space.name };

    return tile;
  }
}
```

### 2. The Token Movement Controller (`entities/TokenController.js`)

When the WebSocket sends a `DICE_ROLLED` event, the token cannot just teleport. It needs to hop step-by-step through the 3D tiles. We use **GSAP** (an animation library) to sequence the jumps so it looks like a physical game piece.

```javascript
import * as THREE from 'three';
import { gsap } from 'gsap';

export class TokenController {
  constructor(scene, boardTiles) {
    this.scene = scene;
    this.boardTiles = boardTiles; // Array of 3D tile meshes
    
    // Create a simple red token for PH
    const geo = new THREE.ConeGeometry(0.8, 2, 8);
    const mat = new THREE.MeshStandardMaterial({ color: 0xff0000 });
    this.token = new THREE.Mesh(geo, mat);
    this.token.position.y = 1; // Hover above board
    this.scene.add(this.token);
  }

  // Called when server says "DICE_ROLLED" with a target index
  async movePlayer(targetTileIndex) {
    const currentTileIndex = this.token.userData.currentTile || 0;
    let stepsToMove = targetTileIndex - currentTileIndex;
    
    // Handle board wraparound (e.g., moving from space 39 to space 5)
    if (stepsToMove < 0) {
      stepsToMove += 40;
    }

    // Create an animation timeline for sequential hops
    const timeline = gsap.timeline();

    for (let i = 1; i <= stepsToMove; i++) {
      const nextTileIndex = (currentTileIndex + i) % 40;
      const targetTile = this.boardTiles[nextTileIndex];
      
      // Add a hop animation to the timeline
      timeline.to(this.token.position, {
        x: targetTile.position.x,
        z: targetTile.position.z,
        duration: 0.3,
        ease: "power1.inOut",
        // Add a "hop" in the middle of the movement
        onUpdate: () => {
          // Simple sine wave for the hop height
          const progress = timeline.progress();
          this.token.position.y = 1 + Math.sin(progress * Math.PI) * 2;
        }
      });
    }

    // Update current position
    this.token.userData.currentTile = targetTileIndex;

    // Return a promise that resolves when animation finishes
    return new Promise(resolve => timeline.then(resolve));
  }
}
```

### Why this architecture matters:
1. **Separation of Concerns:** `BoardBuilder` handles the math of the square. `TokenController` only cares about moving a mesh from point A to point B. Neither knows about the AI or the game rules.
2. **Deterministic Movement:** The server (Durable Object) calculates the *final target tile index* (e.g., space 12). The frontend just says, "Okay, hop from my current space to space 12."
3. **GSAP Timeline:** By chaining the hops in a timeline, the token physically bounces through spaces 6, 7, 8, 9, 10, 11, and 12, rather than gliding in a straight line across the center of the board.
