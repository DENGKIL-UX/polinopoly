   133→Work Log:
   134→- Analyzed stuck-game screenshot with VLM: PH was active but no "AI thinking" indicator — AI turns weren't firing after the human's turn.

The AI checked `phase==='buying'` at 1500ms while phase was still 'moving' (handleLanding hadn't fired), so it skipped the buy branch, and the final `if (phase==='landed'||'playing')` was false → endTurn() never called → game frozen.
   136→- SECOND bug: handleLanding had DUPLICATE AI-buy logic (lines 447-506) that called /api/ai-decision (which 404s) and fell back to random, racing with aiTurn's own buy logic. The AI branch never set phase='buying' (only the human branch did), so aiTurn's buying check never matched for AI players.
   137→- Researched GitHub Monopoly AI (intrepidcoder, itaylayzer, AniketSanghi): cash-multiple buy heuristic, greedy-ROI house building, expert-system rules, simple positional eval function.

   139→Fixes implemented:
   140→
   141→1. NEW MODULE: src/lib/ai-engine.ts — chess-like expert-system AI:
   142→   - evaluatePosition(): positional score = cash (diminishing) + property equity + monopoly bonus + jail penalty − opponent-threat. Lets the AI compare decision branches.
   143→   - decideBuy(): 8-rule expert system — cash buffer, cash-multiple affordability, color-group completion (+80), block-opponent-monopoly (+35), ROI ratio, highway/media count-scaling, dark-blue federal-power bonus, inflation hedge, coalition-personality aggression + random variance. Returns shouldBuy + human-readable reason.
   144→   - decideBuild(): greedy ROI on completed monopolies, even-build rule enforced, dark-blue/green prioritised, aggressive coalitions build more.
   145→   - decideJail(): use Get-Out-Of-Jail card if held; early game with few props = wait; late game with monopolies = pay bail; affordability check.
   146→   - decideAuctionBid(): max-bid = price × (0.8 + personality), 50% premium to complete a monopoly, surprise over-bid 20% of the time.
   147→   - getCoalitionPersonality(): 6 coalition profiles (PH cautious, PN aggressive, BN steady, GPS opportunistic, GRS unpredictable, IND chaotic) with aggressionBonus, buildAggression, variance — creates surprising, non-deterministic, flavourful plays.
   148→
   149→2. game-store.ts — rewrote aiTurn with PHASE-POLLING waits:
   150→   - Replaced fragile fixed delays with waitForPhase(targets, timeout, interval=150ms) that polls until the phase matches. No more desync with the 2.3s move pipeline or the token-hop animation.
   151→   - Structured 8-step flow: jail decision → roll → wait-for-landing → buy (expert) → pay rent → draw card (with recursive buy/rent handling) → auction wait → build houses (expert) → end turn.
   152→   - Safety net: if final phase isn't terminal, endTurn() is called anyway so the game can never stall.
   153→   - Removed duplicate AI-buy logic from handleLanding — AI players now land in 'buying' phase just like humans, and aiTurn's expert system decides buy-vs-skip. This eliminated the /api/ai-decision 404 path entirely.
   154→   - aiAuctionTurn upgraded to use decideAuctionBid() expert function.
   155→   - buildAIContext() helper assembles the AIContext (player + tiles + opponents + market + turn) for the engine.

   210→- Added `minWidth: 0; minHeight: 0; overflow: hidden` to each tile wrapper div to reinforce that tiles must not force the grid to grow.

  1529→  const me = ctx.player;
  1530→  const builds: number[] = [];
  1531→  const personality = getCoalitionPersonality(me.coalitionId);
  1532→  // Lower buffer so AI builds more aggressively (was 200-300)
  1533→  const minBuffer = ctx.turnCount < 10 ? 80 : 150;                     // ← AI build buffer
  1534→  // Track virtual money separately (don't mutate ctx.player)
  1535→  let virtualMoney = me.money;

  1539→  for (const tileId of me.properties) {
  1540→    const t = BOARD_TILES[tileId];
  1541→    if (t?.colorGroup) colorGroups.add(t.colorGroup);
  1542→  }
  1543→
  1544→  const buildableTiles: Array<{ tile: Tile; liveTile: Tile; priority: number }> = [];
  1545→
  1546→  for (const cg of colorGroups) {
  1547→    const groupTiles = BOARD_TILES.filter((t) => t.colorGroup === cg);
  1548→    const ownsAll = groupTiles.every((t) => me.properties.includes(t.id));
  1549→    if (!ownsAll) continue;
  1550→

  1552→      const liveTile = ctx.tiles[t.id];
  1553→      const houses = liveTile?.houses ?? 0;
  1554→      if (houses >= 5) continue; // already hotel
  1555→      if (ctx.mortgagedTiles.includes(t.id)) continue;

  1558→

  1560→      // cheaper tiles first for faster ROI. Boost high-rent groups.
  1561→      let priority = 0;
  1562→      if (t.rent && t.rent[0] && t.housePrice) {
  1563→        priority = (t.rent[houses] ?? 0) / t.housePrice; // ROI of next house
  1564→      }
  1565→      // Dark blue & green get extra priority (high late-game rent)
  1566→      if (cg === 'darkblue') priority *= 1.5;
  1567→      if (cg === 'green') priority *= 1.2;
  1568→      // Aggressive coalitions build more
  1569→      priority *= 1 + personality.buildAggression * 0.3;
  1570→
  1571→      buildableTiles.push({ tile: t, liveTile: liveTile ?? t, priority });
  1572→    }
  1573→  }
  1574→
  1575→  // Sort by priority descending
  1576→  buildableTiles.sort((a, b) => b.priority - a.priority);
  1577→
  1578→  // Even-build rule: don't let any tile get more than 1 ahead of others in its group
  1579→  for (const { tile } of buildableTiles) {
  1580→    if (virtualMoney - (tile.housePrice ?? 0) < minBuffer) break;
  1581→    const liveTile = ctx.tiles[tile.id];
  1582→    const myHouses = liveTile?.houses ?? 0;
  1583→    // Check other tiles in same group aren't more than 1 house behind
  1584→    const groupTiles = BOARD_TILES.filter((t) => t.colorGroup === tile.colorGroup);
  1585→    const minInGroup = Math.min(
  1586→      ...groupTiles.map((t) => ctx.tiles[t.id]?.houses ?? 0),
  1587→    );
  1588→    if (myHouses > minInGroup) continue; // would violate even-build     ← AI ENFORCES EVEN-BUILD
  1589→
  1590→    builds.push(tile.id);
  1591→    // Deduct virtually for the next iteration's buffer check
  1592→    virtualMoney -= tile.housePrice ?? 0;
  1593→  }
  1594→
  1595→  return builds;
  1596→}
