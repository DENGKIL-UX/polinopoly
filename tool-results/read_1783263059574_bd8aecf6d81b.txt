   133→Work Log:

The AI checked `phase==='buying'` at 1500ms while phase was still 'moving' (handleLanding hadn't fired), so it skipped the buy branch, and the final `if (phase==='landed'||'playing')` was false → endTurn() never called → game frozen.
   136→- SECOND bug: handleLanding had DUPLICATE AI-buy logic (lines 447-506) that called /api/ai-decision (which 404s) and fell back to random, racing with aiTurn's own buy logic. The AI branch never set phase='buying' (only the human branch did), so aiTurn's buying check never matched for AI players.

   139→Fixes implemented:
   140→
   141→1. NEW MODULE: src/lib/ai-engine.ts — chess-like expert-system AI:
   142→   - evaluatePosition(): positional score = cash (diminishing) + property equity + monopoly bonus + jail penalty − opponent-threat. Lets the AI compare decision branches.
   143→   - decideBuy(): 8-rule expert system — cash buffer, cash-multiple affordability, color-group completion (+80), block-opponent-monopoly (+35), ROI ratio, highway/media count-scaling, dark-blue federal-power bonus, inflation hedge, coalition-personality aggression + random variance. Returns shouldBuy + human-readable reason.
   144→   - decideBuild(): greedy ROI on completed monopolies, even-build rule enforced, dark-blue/green prioritised, aggressive coalitions build more.
   145→   - decideJail(): use Get-Out-Of-Jail card if held; early game with few props = wait; late game with monopolies = pay bail; affordability check.
   146→   - decideAuctionBid(): max-bid = price × (0.8 + personality), 50% premium to complete a monopoly, surprise over-bid 20% of the time.
   147→   - getCoalitionPersonality(): 6 coalition profiles (PH cautious, PN aggressive, BN steady, GPS opportunistic, GRS unpredictable, IND chaotic) with aggressionBonus, buildAggression, variance — creates surprising, non-deterministic, flavourful plays.
   148→
   149→2. game-store.ts — rewrote aiTurn with PHASE-POLLING waits:
   150→   - Replaced fragile fixed delays with waitForPhase(targets, timeout, interval=150ms) that polls until the phase matches. No more desync with the 2.3s move pipeline or the token-hop animation.
   151→   - Structured 8-step flow: jail decision → roll → wait-for-landing → buy (expert) → pay rent → draw card (with recursive buy/rent handling) → auction wait → build houses (expert) → end turn.
   152→   - Safety net: if final phase isn't terminal, endTurn() is called anyway so the game can never stall.
   153→   - Removed duplicate AI-buy logic from handleLanding — AI players now land in 'buying' phase just like humans, and aiTurn's expert system decides buy-vs-skip. This eliminated the /api/ai-decision 404 path entirely.
   154→   - aiAuctionTurn upgraded to use decideAuctionBid() expert function.
   155→   - buildAIContext() helper assembles the AIContext (player + tiles + opponents + market + turn) for the engine.
