    70→### ?? 3. The Game Phases (State Machine)
    71→
    72→The Durable Object manages the game as a strict state machine to prevent desyncs between Three.js clients.
    73→
    74→1. **Phase 1: Penggal (Session Start)**
    75→   * Players join via WebSocket. Empty slots are assigned to AI Coalitions.
    76→2. **Phase 2: Undi (Roll Dice)**
    77→   * Current player rolls. Three.js animates. Server calculates target tile.
    78→3. **Phase 3: Dewan (Action Resolution)**
    79→   * Evaluate tile: Buy, Pay Rent, or Draw Card.
    80→   * *If AI turn:* Server pauses, calls GLM 5.2, waits for AI decision, executes.
    81→4. **Phase 4: Krisis (Event Phase)**
    82→   * If doubles rolled, trigger GLM 5.2 to generate a "Krisis Nasional" based on real-time market data.
    83→5. **Phase 5: Pembubaran (End Game)**
    84→   * Player reaches 112 Seats (Properties) or bankrupts others.

    88→### ?? 4. GLM 5.2 Agent Integration & Prompt Engineering
    89→
    90→The AI acts as both the "Speaker of the House" (Narrator) and the independent political candidates. We use a structured system prompt to enforce JSON output, making it easy for the Durable Object to parse the AI's decisions.
    91→
    92→```javascript
    93→// backend/src/ai-agent.js
    94→
    95→const GLM_SYSTEM_PROMPT = `
    96→You are the Game Master and AI Political Agent for "Dewan Rakyat", a Malaysian Political Monopoly game.
    97→You simulate cynical, satirical Malaysian politicians. You speak in a mix of formal Malay and Manglish.
    98→
    99→CURRENT WORLD STATE:
   100→- Market (PasarAPI): KLCI is {klcChange}%, CPO is RM{cpoPrice}.
   101→- Seat Demographics (Tindak Malaysia): {demographicBreakdown}
   102→
   103→YOUR ROLE:
   104→1. If asked to decide a move for an AI player, respond with a JSON object.
   105→2. Evaluate the property cost vs AI's current votes. Consider demographic advantages.
   106→3. Provide a 1-sentence satirical quote in Manglish/Malay.
   107→
   108→RESPONSE FORMAT (Strict JSON):
   109→{
   110→  "action": "BUY" | "IGNORE" | "TRADE",
   111→  "quote": "Your satirical quote here.",
   112→  "reasoning": "Brief internal logic."
   113→}
   114→`;
   115→
   116→export async function getAIMove(gameState, env) {
   117→  const marketData = JSON.parse(await env.MARKET_KV.get("market_state"));
   118→  
   119→  const promptPayload = {
   120→    klcChange: marketData.klciChange,
   121→    cpoPrice: marketData.cpo,
   122→    demographicBreakdown: gameState.currentTile.demographics,
   123→    aiVotes: gameState.currentPlayer.votes,
   124→    tilePrice: gameState.currentTile.price
   125→  };
   126→
   127→  const response = await fetch(env.GLM_API_ENDPOINT, {
   128→    method: 'POST',
   129→    headers: {
   130→      'Authorization': `Bearer ${env.GLM_API_KEY}`,
   131→      'Content-Type': 'application/json'
   132→    },
   133→    body: JSON.stringify({
   134→      model: "glm-5.2", // Target GLM 5.2
   135→      messages: [
   136→        { role: "system", content: GLM_SYSTEM_PROMPT.replace(/{(\w+)}/g, (_, k) => promptPayload[k] || '') },
   137→        { role: "user", content: `You landed on ${gameState.currentTile.name}. Do you buy it?` }
   138→      ],
   139→      temperature: 0.7 // Slightly creative but logical
   140→    })
   141→  });
   142→
   143→  const data = await response.json();
   144→  
   145→  try {
   146→    // Extract JSON from GLM response
   147→    const content = data.choices[0].message.content;
   148→    const jsonStart = content.indexOf('{');
   149→    const jsonEnd = content.lastIndexOf('}');
   150→    return JSON.parse(content.substring(jsonStart, jsonEnd + 1));
   151→  } catch (e) {
   152→    // Fallback if GLM hallucinates
   153→    return { action: "IGNORE", quote: "Macham, saya kenot brain sekarang.", reasoning: "Parse error" };
   154→  }
   155→}

   160→### ?? 5. Authoritative Game State (Durable Object)
   161→
   162→This is the brain of the multiplayer game. It handles WebSocket connections, processes turns, and calls the AI agent.
   163→
   164→```javascript
   165→// backend/src/GameDurable.js
   166→
   167→import { getAIMove } from './ai-agent.js';
   168→
   169→export class GameDurable {
   170→  constructor(state, env) {
   171→    this.state = state;
   172→    this.env = env;
   173→    this.sessions = new Map(); // WebSocket clients
   174→  }
   175→
   176→  async fetch(request) {
   177→    const pair = new WebSocketPair();
   178→    const [client, server] = pair;
   179→
   180→    this.handleSession(server);
   181→    return new Response(null, { status: 101, webSocket: client });
   182→  }
   183→
   184→  async handleSession(websocket) {
   185→    this.state.acceptWebSocket(websocket);
   186→    this.sessions.add(websocket);
   187→
   188→    websocket.addEventListener('message', async (event) => {
   189→      const msg = JSON.parse(event.data);
   190→      
   191→      if (msg.type === 'ROLL_DICE') {
   192→        await this.processTurn(msg.playerId);
   193→      }
   194→    });
   195→  }
   196→
   197→  async broadcast(message) {
   198→    const msgStr = JSON.stringify(message);
   199→    for (const ws of this.sessions) {
   200→      ws.send(msgStr);
   201→    }
   202→  }
   203→
   204→  async processTurn(playerId) {
   205→    const dice1 = Math.floor(Math.random() * 6) + 1;
   206→    const dice2 = Math.floor(Math.random() * 6) + 1;
   207→    const total = dice1 + dice2;
   208→    
   209→    // Update position
   210→    const player = this.gameState.players[playerId];
   211→    player.position = (player.position + total) % 40;
   212→    const tile = this.gameState.board[player.position];
   213→
   214→    await this.broadcast({ type: 'DICE_ROLLED', dice: [dice1, dice2], playerId });
   215→
   216→    // If player is AI, ask GLM 5.2 what to do
   217→    if (player.isAI) {
   218→      // Broadcast "AI is thinking..."
   219→      await this.broadcast({ type: 'AI_THINKING', playerId });
   220→      
   221→      const aiDecision = await getAIMove({
   222→        currentPlayer: player,
   223→        currentTile: tile
   224→      }, this.env);
   225→
   226→      if (aiDecision.action === 'BUY' && player.votes >= tile.price) {
   227→        player.votes -= tile.price;
   228→        tile.owner = playerId;
   229→      }
   230→
   231→      await this.broadcast({ 
   232→        type: 'AI_ACTION', 
   233→        playerId, 
   234→        action: aiDecision.action, 
   235→        quote: aiDecision.quote 
   236→      });
   237→    }
   238→
   239→    // Check for doubles or pass to next player
   240→    if (dice1 !== dice2) {
   241→      this.gameState.currentTurn = (this.gameState.currentTurn + 1) % this.gameState.players.length;
   242→    }
   243→
   244→    await this.broadcast({ type: 'STATE_UPDATE', state: this.gameState });
   245→  }
   246→}
   247→```

   360→### 1. The 3D Board Generator (`scene/BoardBuilder.js`)
   361→
   362→This module calculates the (x, z) coordinates for all 40 spaces in a square loop and renders them as 3D meshes.
   363→
   364→```javascript
   365→import * as THREE from 'three';
   366→
   367→// The 40 spaces (simplified for brevity, mapping to previous JSON)
   368→const BOARD_SPACES = [
   369→  { id: 0, name: 'Pilihan Raya', type: 'go', color: 0x00FF00 },
   370→  { id: 1, name: 'MUDA', type: 'property', color: 0x8B4513 },
   371→  // ... spaces 2 to 38 ...
   372→  { id: 39, name: 'Kuala Lumpur', type: 'property', color: 0x000080 }
   373→];
   374→
   375→export class BoardBuilder {
   376→  constructor(scene) {
   377→    this.scene = scene;
   378→    this.tiles = [];
   379→    this.tileSize = 4; // Size of each square
   380→    this.boardWidth = 11 * this.tileSize; // 11 tiles per side
   381→  }
   382→
   383→  build() {
   384→    const boardGroup = new THREE.Group();
   385→
   386→    BOARD_SPACES.forEach((space, index) => {
   387→      const tileMesh = this.createTile(space, index);
   388→      boardGroup.add(tileMesh);
   389→      this.tiles.push(tileMesh);
   390→    });
   391→
   392→    // Add center base
   393→    const baseGeo = new THREE.BoxGeometry(this.boardWidth, 0.5, this.boardWidth);
   394→    const baseMat = new THREE.MeshStandardMaterial({ color: 0x1a1a1a });
   395→    const base = new THREE.Mesh(baseGeo, baseMat);
   396→    base.position.y = -0.3;
   397→    boardGroup.add(base);
   398→
   399→    this.scene.add(boardGroup);
   400→    return this.tiles;
   401→  }
   402→
   403→  // Calculate (x, z) for a tile in a square loop
   404→  getTilePosition(index) {
   405→    const step = this.tileSize;
   406→    const halfBoard = this.boardWidth / 2;
   407→    const halfStep = step / 2;
   408→    const tilesPerSide = 10; // 10 non-corner tiles per side + 4 corners = 11 per side
   409→
   410→    let x, z;
   411→
   412→    if (index < tilesPerSide + 1) {
   413→      // Side 1: Right side (Bottom to Top) - Spaces 0 to 10
   414→      x = halfBoard - halfStep;
   415→      z = halfBoard - halfStep - (index * step);
   416→    } else if (index < (tilesPerSide * 2) + 1) {
   417→      // Side 2: Top side (Right to Left) - Spaces 11 to 20
   418→      x = halfBoard - halfStep - ((index - (tilesPerSide + 1)) * step);
   419→      z = -halfBoard + halfStep;
   420→    } else if (index < (tilesPerSide * 3) + 1) {
   421→      // Side 3: Left side (Top to Bottom) - Spaces 21 to 30
   422→      x = -halfBoard + halfStep;
   423→      z = -halfBoard + halfStep + ((index - (tilesPerSide * 2 + 1)) * step);
   424→    } else {
   425→      // Side 4: Bottom side (Left to Right) - Spaces 31 to 39
   426→      x = -halfBoard + halfStep + ((index - (tilesPerSide * 3 + 1)) * step);
   427→      z = halfBoard - halfStep;
   428→    }
   429→
   430→    return { x, z };
   431→  }
   432→
   433→  createTile(space, index) {
   434→    const pos = this.getTilePosition(index);
   435→    
   436→    const geometry = new THREE.BoxGeometry(this.tileSize * 0.95, 0.2, this.tileSize * 0.95);
   437→    const material = new THREE.MeshStandardMaterial({ 
   438→      color: space.color || 0x333333,
   439→      roughness: 0.6 
   440→    });
   441→    
   442→    const tile = new THREE.Mesh(geometry, material);
   443→    tile.position.set(pos.x, 0, pos.z);
   444→    tile.receiveShadow = true;
   445→    tile.userData = { spaceId: index, name: space.name };
   446→
   447→    return tile;
   448→  }
   449→}
   450→```

   452→### 2. The Token Movement Controller (`entities/TokenController.js`)
   453→
   454→When the WebSocket sends a `DICE_ROLLED` event, the token cannot just teleport. It needs to hop step-by-step through the 3D tiles. We use **GSAP** (an animation library) to sequence the jumps so it looks like a physical game piece.
   455→
   456→```javascript
   457→import * as THREE from 'three';
   458→import { gsap } from 'gsap';
   459→
   460→export class TokenController {
   461→  constructor(scene, boardTiles) {
   462→    this.scene = scene;
   463→    this.boardTiles = boardTiles; // Array of 3D tile meshes
   464→    
   465→    // Create a simple red token for PH
   466→    const geo = new THREE.ConeGeometry(0.8, 2, 8);
   467→    const mat = new THREE.MeshStandardMaterial({ color: 0xff0000 });
   468→    this.token = new THREE.Mesh(geo, mat);
   469→    this.token.position.y = 1; // Hover above board
   470→    this.scene.add(this.token);
   471→  }
   472→
   473→  // Called when server says "DICE_ROLLED" with a target index
   474→  async movePlayer(targetTileIndex) {
   475→    const currentTileIndex = this.token.userData.currentTile || 0;
   476→    let stepsToMove = targetTileIndex - currentTileIndex;
   477→    
   478→    // Handle board wraparound (e.g., moving from space 39 to space 5)
   479→    if (stepsToMove < 0) {
   480→      stepsToMove += 40;
   481→    }
   482→
   483→    // Create an animation timeline for sequential hops
   484→    const timeline = gsap.timeline();
   485→
   486→    for (let i = 1; i <= stepsToMove; i++) {
   487→      const nextTileIndex = (currentTileIndex + i) % 40;
   488→      const targetTile = this.boardTiles[nextTileIndex];
   489→      
   490→      // Add a hop animation to the timeline
   491→      timeline.to(this.token.position, {
   492→        x: targetTile.position.x,
   493→        z: targetTile.position.z,
   494→        duration: 0.3,
   495→        ease: "power1.inOut",
   496→        // Add a "hop" in the middle of the movement
   497→        onUpdate: () => {
   498→          // Simple sine wave for the hop height
   499→          const progress = timeline.progress();
   500→          this.token.position.y = 1 + Math.sin(progress * Math.PI) * 2;
   501→        }
   502→      });
   503→    }
   504→
   505→    // Update current position
   506→    this.token.userData.currentTile = targetTileIndex;
   507→
   508→    // Return a promise that resolves when animation finishes
   509→    return new Promise(resolve => timeline.then(resolve));
   510→  }
   511→}
   512→```

   562→### Why this architecture matters:
   563→1. **Separation of Concerns:** `BoardBuilder` handles the math of the square. `TokenController` only cares about moving a mesh from point A to point B. Neither knows about the AI or the game rules.
   564→2. **Deterministic Movement:** The server (Durable Object) calculates the *final target tile index* (e.g., space 12). The frontend just says, "Okay, hop from my current space to space 12."
   565→3. **GSAP Timeline:** By chaining the hops in a timeline, the token physically bounces through spaces 6, 7, 8, 9, 10, 11, and 12, rather than gliding in a straight line across the center of the board.
